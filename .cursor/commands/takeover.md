---
description: 接手现有代码库
---

执行中途接手流程，快速建立对代码库的认知。

流程分五步：读取交接文档、检查 AGENT.md、通读代码库、更新或创建 AGENT.md、补齐缺失文档。

1. 检查 `HANDOVER.md`：如果存在，读取上一个 AGENT 留下的工作摘要、注意事项和后续建议，读取后删除
2. 检查 `AGENT.md`：如果存在，读取项目概况、关键线索和常犯错误；如果不存在，在通读代码后创建
3. 通读代码库：按入口、核心流程、边界的顺序阅读代码库。入口是启动点、主循环或路由入口，核心流程是数据从输入到处理再到输出的流转路径，边界是与外部系统交互的地方。代码库较大时可调用子 agent 并行探索不同部分。
4. 更新 `AGENT.md`：发现信息不一致或遗漏时更新
5. 补齐缺失文档：如果缺少 `spec.md`、`design.md` 或 `plan.md`，询问用户是否补齐

需要开发新功能时，按 `spec_driven.mdc` 流程处理。
